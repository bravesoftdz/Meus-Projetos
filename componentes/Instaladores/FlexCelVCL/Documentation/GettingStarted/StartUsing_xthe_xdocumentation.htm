Using the documentation
The first place to visit when you are starting in something, is of course the documentation,
so this is where we will start too.
FlexCel documentation is divided in three sections:
Concepts
Those are the xls pdf documents at the "documentation" folder. (not the FlexCel API Reference)
They provide the basic concepts you need to know before starting to code,
and you are encouraged to start with them.
The documents are kept as short and concise as possible, in order to
not overload them, and have the most important information easily accessible
You can access them from the Start Menu:
Demos
Demos are a very important part of the FlexCel documentation as they answer two questions at
the same time: "What can I do?"  and "How can I do it?"
It is our experience that they provide the best way to answer both questions.
Instead of a 500 page pdf with code, we actually provide you compiling code
that you can debug, step in and modiy.
To reflect the double function of the demos, they can be used in two different ways.
For the times when you want to know what can be done, we provide a demo browser, the FlexCel Well,
that you can access from the start menu.
From there you can easily navigate through all the available demos, reading a description on every one of them
Now, when you already know what you want to do, and want to know how to do it, you can open
the demo as a separate application, and run it alone without the well.
All FlexCel demos are written in a way that they can run inside the well, or as standalone applications.
Reference
Besides the conceptual documentation in pdfs and the hands on documentation in the demos,
we also provide reference for every method in FlexCel, so you can know what it does.
The easiest way is to just press F1 inside the IDE. This works in all Delphi and C++ Ides.
If the help has been registered at install time, it will be available when you press F1 over a FlexCel class or method.
