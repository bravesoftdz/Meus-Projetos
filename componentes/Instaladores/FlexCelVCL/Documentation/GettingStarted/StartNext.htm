Where to go next
This concludes this small guide.
We believe that the best documentation is "no documentation needed"; that is,
whenever something is too difficult to understand that it needs too much
documentation, documenting it should be the last resource. The first is to
try to simplify the problem so it works the way you expect it to, and you
can do what you want to do without consulting the documentation.
But there are cases where there is no escape, and in complex libraries
like FlexCel there is always going to be a learning curve, no matter
how much we do to make it smaller. So take your time to get used to it.
From here, the next steps are to read the documentation, or just run the demos
in order to learn what can be done.
Thanks for choosing FlexCel.
