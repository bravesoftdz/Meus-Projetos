Using the components
FlexCel components are of two different types:
Non visual components
Most FlexCel components are non visual, and you can drop them on a form or just create them by code.
Note that there are situations where you can not drop them in form, so you will need to create them yourself.
TFlexCelImport:   It contains the API to read and write xls files.
A FlexCelImport component conneted to a TXLSAdapter component
is all you need to create or read Excel files.
TFlexCelReport: FlexCelReport is a component designed to create Excel files.
It reads a template Xls file, fills in the blanks with values from memory or the database, and writes them back
TFlexCelReportNoDB: See it as a "light" FlexCelReport.
If your application doesn’t use databases you should use this component instead
of FlexCelReport to avoid DB overhead. If you are using DB’s in any way, you just don’t need it.
TFlxMemTable: It acts as an "adapter" to feed table data to a FlexCelReport.
If your data is not in a database, and it is for example in a TList, this is
the component you need to use this data with FlexCelRerport.
TXlsTemplateStore: Allows you to embed the xls files used by FlexCelReport (Or even FlexCelImport)
into the exe file of your application. This way you can avoid distributing additional files.
When you connect this component to an XlsAdapter, the XlsAdapter will read the files from there.
TXlsAdapter: This is the engine actually doing the work behind FlexCelImport or FlexCelReport.
It can also centralize many options for all the FlexCelImport/FlexCelReport components
connected to it. (like the template store they all will use). You only need to drop one
XlsAdapter into the form, and you can connect as many FlexCelReport/Import components as you wish.
Visual Components
There are currently two components in this category:
TFlexCelGrid:   This is an "Excel-like" grid that can display and edit your xls files.
You can also use it to natively print xls files.
TFlexCelPreview:   If you connect it to a FlexCelGrid, you can  preview what is going to be printed.
