New on v 3.2

Excel 2010 XLS file support. Excel 2010 introduced a new "Protected View" that will flag old FlexCel xls files as "unsafe". This release fixes this so files will open in 2010 without warnings. It is important to update to this version as soon as possible, so when Excel 2010 is released your files will keep on working without warnings.



New on v 3.1.1

Improved import of currency and percent values. Now when loading values like "5%" or "$ 1.2" from a csv file, or when entering those values with SetCellString, the cells will be formatted to reflect the format in the original string.

New on v 3.0

Rad Studio 2009 Support. This new version is optimized to use the new Delphi 2009 features when present, including generics and UnicodeString instead of widestring.

Lazarus/FPC Support. FlexCel now can be installed in Lazarus, and run in Linux or macs.

New Native Xls filesystem. The xls filesystem has been fully rewriten in native pascal code, making it faster and also usable from any Operating System without any third party dlls.

New FlexCel Designer. The new FlexCelDesigner doesn't embed Excel as the previous version did. This was necessary because Excel 2007 doesn't allow embedding by default, but also makes the designer much more robust.

Completely rewritten Documentation and Demos. All documents and demos have been rewritten to provide an easier learning curve. Now documentation is similar to .NET, with a Demo Browser, PDF reference files and a complete Help Reference for all methods and classes.

Easier Setup. All source files are now stored in one folder, so there is only need to add one path to the library path. A new Setup.exe will create everything automatically.

Lots of small improvements and bug fixes. Added support for Autofilters in FlexCelImport. FlexCelReport can now run a report from a stream. New methods ToOADate and FromOADate to make easier to work with 1904 dates. Syntax "=A:B" or "=1:2" to reference full rows or columns is now supported when entering formulas. And much more.

Breaking Changes. Changes noted below are breaking from 2.6, so please take a look at them when upgrading.

Changed Add/AssignPicture Methods to use Array of bytes instead of strings for data. These methods used a string to hold data because at the time they were created Delphi didn't support dynamic arrays. But now with Delphi 2009 a string is not anymore a safe place to store binary data, so those methods now use Array of bytes instead. When changing code remember that strings start a position [1], and Arrays of bytes at position [0].

Deprecated OLEAdapter. OLEAdapter is not included anymore with FlexCel, and you should replace it with XlsAdapter. OLEAdapter used OLE Automation to create the files, and that was not reliable, creating a lot of support issues. XlsAdapter is faster and more reliable. FlexCel doesn't have any non-native code support now.

Upgraded TemplateStore. TemplateStore now uses the new native filesystem. You need to refresh the TemplateStores you have in order to use them with FlexCel 3.0.

Deprecated DOAFlexCelReport and IBOFlexCelReport. Those components added no functionality to FlexCelReport, so now you should replace them with FlexCelReport.

Deprecated DBTemplateStore. This component was confusing to use and didn't add much value. To run reports from a database, simply use the new Run(InStream, OutStream) method, using a stream with the data from the DataBase as InStream.

Deprecated CLX. Since Kylix has not been updated for too many years and it doesn't even install in modern Linux distributions, we are deprecating CLX support. We offer Lazarus support now which we believe is a much better option at this time for multi-platform.

Deprecated Delphi5 /BCB5 support. In order to concentrate in newer delphi releases and using new features, we have to stop supporting old versions. Delphi5/BCB5 packages are not included anymore.
